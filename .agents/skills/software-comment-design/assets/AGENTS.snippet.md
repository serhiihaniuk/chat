## Comment quality

When adding, reviewing, or modifying code comments, use the `$software-comment-design` skill.
Prefer comments that capture contracts, invariants, rationale, edge cases, and non-guarantees.
Delete comments that only repeat code. Do not invent design rationale that is not supported by code, tests, or explicit context.
