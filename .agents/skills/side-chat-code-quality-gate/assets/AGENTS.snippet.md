## Repository code quality gate

Use the local code-quality skill when writing, reviewing, or refactoring TypeScript, Node, or React code.

It covers the configured formatter, linter, strict type checks, governance checks, package boundaries, cognitive complexity, source budgets, framework/SDK readability, and comment quality. Discover the actual tools and thresholds from the repository before applying it.

Treat “clever but hard to understand” as a quality failure even when code passes typecheck. The target is human-level cognitive load, not AI-level comprehension. Comments must lower context requirements; they should not assume the reader already knows the whole architecture.

For test-specific work, also use the local testing-architecture skill.
