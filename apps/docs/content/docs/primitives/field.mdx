Field is a thin role wrapper over Base UI `Field`. `Field.Label`, `Field.Description`, and
`Field.Error` carry the typographic roles, while `Field.Control` owns the input/textarea
chrome — `--field-bg`, `--field-border`, `--field-radius`, and the `--field-px` / `--field-py`
padding. Clicking the label focuses the control automatically: Base UI wires the association,
so the markup never hand-writes `htmlFor`/`id`. Validation surfaces through `Field.Error` and
the `invalid:` variant, which swaps the control border to `--destructive`.

## Usage

```tsx
import { Field } from '@base-ui/react/field';

<Field.Root>
  <Field.Label>Custom instructions</Field.Label>
  <Field.Description>Shared with the model.</Field.Description>
  <Field.Control value={value} onValueChange={setValue} render={<input />} />
  <Field.Error match>This field is required.</Field.Error>
</Field.Root>
```

Render a `<textarea>` instead of an `<input>` through the `render` prop for multi-line input.
Set `invalid` on `Field.Root` (or let Base UI drive it from validation) to arm the
destructive border and reveal the error text.
