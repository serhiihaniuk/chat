Model selector is a Base UI `Combobox` that sits in the composer footer. Its trigger reads
`Model / Thinking` with a chevron; the popup (`data-slot="combobox-content"`, sized to
`w-menu`) stacks a `Search` input, the searchable model list, a `Segmented` thinking-level
control, and a status line summarising the current choice. Each model row pairs a `Media`
glyph with a name + description and a `Check` indicator on the selected item. Selection and
thinking level are controlled-or-uncontrolled, and the popup portals into the widget root via
`usePortalContainer()` so it stays inside the panel.

## Usage

```tsx
import { ModelSelector, type Model } from '@side-chat/side-chat-widget/ui/model-selector';

const models: Model[] = [
  { id: 'sonnet', name: 'Claude Sonnet', desc: 'Balanced — everyday tasks', icon: <Sparkles size={16} /> },
  { id: 'opus', name: 'Claude Opus', desc: 'Deepest reasoning, slower', icon: <Brain size={16} /> },
];

<ModelSelector
  models={models}
  defaultModel={models[0]}
  defaultThinking="medium"
  onValueChange={setModelId}
  onThinkingChange={setThinking}
/>
```

Omit `value` / `thinkingValue` to let the selector own its state, or pass them (with the
matching `onValueChange` / `onThinkingChange`) to drive it. `thinkingLevels` defaults to
Low / Medium / High but can be overridden.
