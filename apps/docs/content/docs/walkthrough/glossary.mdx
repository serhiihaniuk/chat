## AI / LLM words

See [AI Vocabulary](/docs/walkthrough/vocabulary) for the full treatment.

| Term | One-line meaning |
| --- | --- |
| LLM | A stateless text-predictor. It remembers nothing between calls, so the server resends the whole transcript every turn. |
| Token | The chunk of text an LLM counts (~Âľ of a word). You pay per token and every model has a hard limit. |
| Context window | The maximum tokens (history + new message + reply) a model accepts in one call. |
| Prompt | The text you send the model â€” in practice a list of messages, each tagged with a role. |
| Role | `system` (server's standing rules), `user` (the human), `assistant` (the model's past replies). The browser may set none of them. |
| Streaming / delta | Pushing the reply fragment-by-fragment as it is generated; each fragment is a delta. The UI concatenates them. |
| SSE | Server-Sent Events â€” a one-way HTTP stream (server â†’ browser) carrying the deltas and activities. |
| Tool / function-calling | A function you let the model request. The model cannot act â€” it asks, your code runs it, you feed the result back. |
| Tool loop | Model asks â†’ your code runs the tool â†’ result fed back â†’ model continues, repeating until it can answer. |
| Provider vs model | Provider = the API you call (only OpenAI is real here, plus a fake test provider). Model = an id like `gpt-5.4-mini`. |
| Reasoning | "Thinking" tokens a model spends privately before answering; hidden from the user by default. |
| Agentic | The system can take actions (call tools), observe results, and loop â€” not just reply once. |
| Turn | One user message in â†’ one full assistant response out (including any tool loops). Recorded as one `assistant_turn` row. |

## Architecture & pattern words

See [Patterns in the Code](/docs/walkthrough/patterns).

| Term | One-line meaning |
| --- | --- |
| Port | An interface the core needs but does not implement (`ClockPort`, `AiRuntimePort`). |
| Adapter | A concrete object that fulfills a port using a real library (OpenAI, Postgres, HTTP). |
| Dependency inversion | Infrastructure depends on an interface the core owns â€” not the other way around. |
| Composition root | The one function (`composePartnerAiService`) that picks real adapters and wires them into the ports at startup. |
| Dependency injection | Passing a function what it needs as arguments instead of letting it reach for a global. |
| Factory | A `create*` function that turns typed config into one finished, validated bundle (e.g. `createServiceToolBundle`). |
| Strategy | A swappable behavior expressed as a function-shaped type and injected (the `SystemPromptBuilder`). |
| Registry | A boot-time validator that checks declarations and returns a frozen, name-indexed collection; fails loud on a bad name. |
| State machine | The stream's allowed transitions, so a wrong sequence is impossible â€” `protocol-stream-state-machine.ts` (`advanceProtocolStream`). |

## TypeScript / library gotchas

| Term | One-line meaning |
| --- | --- |
| Effect | The async library used instead of `Promise`. `Effect.Effect<A, E>` = a lazy Promise of `A` that fails with typed `E`. |
| Stream | Effect's streaming type. `Stream.Stream<A, E>` = a sequence of `A`s that may fail with `E` (the protocol event stream). |
| Fiber | Effect's lightweight thread. Generation runs on a server-owned fiber that one instance owns and that outlives the HTTP request. |
| Branded type | `Brand<string, "RequestId">` â€” a plain string at runtime, with a compile-time label so ids cannot be mixed up. |
| Terminal event | The one event that closes a stream: `sidechat.completed`, `sidechat.error`, or `sidechat.blocked`. Nothing may follow it. |
| Finish reason | Why a completed turn ended: `stop` (normal), `length` (hit token limit), `aborted` (cancelled). |
