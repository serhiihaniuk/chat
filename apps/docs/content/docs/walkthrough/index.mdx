This walkthrough follows one user message through the real Side Chat code. It introduces each idea
before naming its API. Use [System Design](/docs/system-design) for terse architecture contracts and
the [Design System](/docs/design-system) for the live UI surface.

## How to read this guide

- Start with **Mental Model** if the package layout is unfamiliar.
- Start with **AI Vocabulary** if model, token, delta, or tool-loop language is unfamiliar.
- Use **Life of a Turn** to connect all layers in one ordered trace.
- Use **Navigating the Code** when you already have a concrete change to make.
