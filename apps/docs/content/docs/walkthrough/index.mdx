A plain-English, example-first tour of how this AI chat system is built — for engineers who know
JavaScript and UI well, but are new to building AI agents. We don't just name the pieces; for
every concept we show how it's actually wired in the code.

This walkthrough is the **onboarding** companion to the reference docs. The
[Design System](/docs/design-system) documents the UI, and [System Design](/docs/system-design)
is the precise architecture reference; this section is the narrative that makes them land. It was
produced by reading the real code in this repo — so when this guide and the code ever disagree,
**trust the code**.

## How to read this guide

Each section assumes the vocabulary from the ones before it, so the words always have somewhere to
land. But you can jump in based on what you need:

- **New to AI backends?** Start at [AI Vocabulary](/docs/walkthrough/vocabulary) — it assumes nothing.
- **Get the concepts, but can't picture the code?** Each pattern and term is given the idea first,
  then how it's actually built.
- **Want to see it all click together?** *Life of a Turn* traces one real message end to end, and
  *Add a Tool* has you wire up a real tool by hand.
- **Just need the architecture map?** [System Design](/docs/system-design) lists every package and
  its one job.
