Text buttons are pure tier-1 utilities on a native `<button>`, so `hover:` and
`focus-visible:` are allowed (the no-`:hover` rule only applies to Base UI parts). Icon
buttons are a square hit-target carried by the `sc-icon-btn` hook class.

## Variants

- **primary** - one per view; the single filled action (`--primary`).
- **secondary** - bordered muted/secondary surface; used by the rail "New chat" control and other paired actions.
- **ghost** - transparent until hovered (`--accent`); for low-emphasis actions.
- **outline** - bordered card surface for toolbars.
- **IconButton** - square, `--size-control` (2rem); **requires an `aria-label`**.

## Anatomy & states

```tsx
import { Button, IconButton } from '@side-chat/side-chat-widget/ui/button';

<Button variant="secondary">New chat</Button>
<IconButton aria-label="Settings"><Settings size={16} /></IconButton>
```

`disabled` drops opacity to 50% and removes pointer events. Focus shows a 2px
`--ring` outline. Hit area is at least `--size-touch` (2.75rem) on touch.
