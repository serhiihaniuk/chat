import { SeparatorDemo } from '@/components/demos/separator';

The separator is a thin wrapper over Base UI `Separator`, so the rule announces `role="separator"`
with a matching `aria-orientation` instead of being an inert styled `<div>`. The wrapper just
defaults the orientation-driven sizing — `data-horizontal:h-px data-horizontal:w-full` for a
horizontal line and `data-vertical:w-px data-vertical:self-stretch` for a vertical one — so the
common case stays one line. Colour comes from the `bg-border` token; consumers add their own
margin (`my-1.5`, `mx-1.5`) where the rule sits.

## Usage

```tsx
import { Separator } from '@side-chat/side-chat-widget/ui/separator';

<Separator orientation="horizontal" className="my-1.5 bg-border" />
<Separator orientation="vertical" className="mx-1.5 self-stretch bg-border" />
```
