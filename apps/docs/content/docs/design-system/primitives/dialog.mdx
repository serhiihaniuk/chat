Dialog is built from Base UI `Dialog` — `Root`, `Portal`, `Backdrop`, `Popup`, `Title`, `Description`, and `Close` — composed once as `WidgetDialog`. It is PANEL-SCOPED: both the backdrop and the popup are portaled into the widget root via `usePortalContainer()` and positioned absolute, so the modal overlays the widget panel (never the embedding host page) and the backdrop clips to the panel's rounded corners. Surface and motion are owned by styles.css through the `dialog-backdrop` / `dialog-content` slots; the popup has no dedicated palette — it inherits the menu surface (`--popover`, `--popover-foreground`, `--border`, `--shadow-popover`).

`WidgetDialog` is controlled-only (`open` / `onOpenChange`, no Trigger part) because the widget's dialogs open from imperative flows — a clicked markdown link, a destructive confirm — not from a co-located trigger button. The close affordance and title/description slots are built in; actions come in as children. The markdown link-safety confirm (every external link in assistant output) is the shipped consumer, with its copy in the labels bag (`linkSafetyTitle`, `linkSafetyDescription`, `linkSafetyCopy`, `linkSafetyOpen`, `dialogClose`) so it rebrands and localizes with the rest of the widget.

## Usage

```tsx
import { WidgetDialog } from '@side-chat/side-chat-widget/ui/dialog';

const [open, setOpen] = useState(false);

<WidgetDialog
  open={open}
  onOpenChange={setOpen}
  title="Clear this conversation?"
  description="The messages are removed from this device. This cannot be undone."
>
  <div className="mt-4 flex justify-end gap-2">
    <Button variant="ghost" onClick={() => setOpen(false)}>Cancel</Button>
    <Button variant="primary" onClick={clear}>Clear</Button>
  </div>
</WidgetDialog>
```
