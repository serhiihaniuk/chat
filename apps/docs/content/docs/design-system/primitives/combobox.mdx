Combobox is the one selector with a filter input, driven directly by Base UI `Combobox`. There is no manual query state: the built-in fuzzy filter sets `data-highlighted` on the match, and `Combobox.Empty` renders the no-results row (collapsed to nothing via `sc-combo-empty` while it holds matches). Rows reuse the Row + `sc-media` patterns; the selected item shows its `Combobox.ItemIndicator` check. The popup portals into the widget root with `container={container}` and is skinned through the `data-slot="combobox-content"` layer.

## Usage

```tsx
import { Combobox } from '@base-ui/react/combobox';
import { usePortalContainer } from '@side-chat/side-chat-widget/ui/widget-root';

const container = usePortalContainer();

<Combobox.Root items={models} value={model} onValueChange={setModel}>
  <Combobox.Trigger className="sc-icon-btn w-full justify-between px-3">
    <Combobox.Value>{(m) => m?.name ?? 'Select model'}</Combobox.Value>
  </Combobox.Trigger>
  <Combobox.Portal container={container}>
    <Combobox.Positioner side="bottom" align="start" sideOffset={8}>
      <Combobox.Popup data-slot="combobox-content" className="w-menu max-w-full">
        <Combobox.Input placeholder="Search models…" />
        <Combobox.Empty className="sc-combo-empty">No models found.</Combobox.Empty>
        <Combobox.List>{(m) => <Combobox.Item value={m}>{m.name}</Combobox.Item>}</Combobox.List>
      </Combobox.Popup>
    </Combobox.Positioner>
  </Combobox.Portal>
</Combobox.Root>
```
