Citations attribute an answer to its sources. The full set lives under a foldable "N sources" row below the answer — the same Base UI `Collapsible` primitive as the Reasoning fold, with an identical trigger-plus-chevron contract and the `--collapsible-panel-height` animation. Only the panel contents differ: source rows instead of thought and tool rows.

Each source row carries a leader glyph, the source title with its domain as a meta line, and a trailing number chip. A web source's glyph takes a per-source brand colour derived from its domain, standing in for a favicon — the one sanctioned non-token value, since the protocol carries no real favicon. A terminal source with no domain keeps a neutral tokenized quote glyph.

Linking is a separate axis from source kind. A source with a `url` renders as a real anchor: pointer cursor, hover fill, and a trailing open-externally arrow. A terminal source — a pasted excerpt with no destination — drops all three: default cursor, no hover, no arrow. It still lists and still numbers.

Inline markers are the second half. A `[n]` chip (`InlineCitation`, a Base UI `PreviewCard`) sits in the prose right after the claim it supports. Hovering it opens a preview card — the favicon and domain on top, the title, then the model's exact quoted excerpt — while the marker itself takes its dark active state (`--cite-marker-active-*`), pairing the two. The card is portaled into the widget root, so it escapes the message's overflow while staying inside the themed subtree.

Both halves bind by number, and the model authors both. The assistant writes GitHub-flavored Markdown footnotes — `[^1]` markers in the prose and `[^1]: Title — https://url — "exact quote"` definitions at the end — so one author assigns one numbering to the markers and the definitions together; there is no separate channel to drift against. The parser lifts the trailing quote into the source's `excerpt` (the card's snippet); a definition without one still renders, just without the quote line. The widget parses the definitions into sources, renders each `[^n]` marker as its chip, and swaps Streamdown's default footnotes block for the fold. A marker the model wrote without a matching definition degrades to plain text — never a chip pointing at a missing or wrong source.

Spacing and sizing follow density-driven tokens, not hardcoded pixels: the number chip and inline marker size from `--cite-marker-size` / `--cite-marker-px`, the hover card from `--cite-card-pad` and stacks its favicon row, title, and excerpt by `--cite-card-gap`, and the fold sits under the answer at the message's shared `--message-stack-gap` — the same gap that separates the reasoning trace from the answer, so all three read as one rhythm that scales with the Density control. Each of those tokens resolves through `--spacing`, so a denser or roomier setting moves them together.

## Usage

The widget wires this automatically inside `MarkdownContent`: an assistant answer that uses footnote syntax renders inline chips over the fold, with no per-message wiring. The kit parts are exported for a host that renders its own citation surface:

```tsx
import {
  InlineCitation,
  SourcesFold,
  type CitationSource,
} from '@side-chat/side-chat-widget/ui/activity/citations';

const sources: CitationSource[] = [
  // linked, with an excerpt shown in the hover card
  {
    label: 'Regulatory framework on AI',
    url: 'https://ec.europa.eu/ai',
    excerpt: 'The AI Act entered into force on 1 August 2024.',
  },
  { label: '"…fully applicable two years later" — pasted context' }, // terminal
];

// Inline marker: a hover chip bound to one source by its number.
<>The framework applies union-wide<InlineCitation number={1} source={sources[0]} />.</>

// The full set, foldable, under the answer.
<SourcesFold sources={sources} />
```
