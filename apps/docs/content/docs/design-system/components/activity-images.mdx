Activity images render pictures a turn produced — a generated chart, a fetched preview — as inline thumbnails under the answer. The defaults are safe by design: the widget constrains size (`max-h-40`, full-width cap), embeds the payload as a `data:` URI built from the protocol's base64 payload and media type (no host-controlled URL fetch), and no host CSS reaches the `img`. A caption, when present, rides under the thumbnail in meta type.

## Usage

```tsx
import { ActivityImages, type ActivityImageData } from '@side-chat/side-chat-widget/ui/activity/activity-images';

const images: ActivityImageData[] = [
  {
    alt: 'Generated chart preview',
    caption: 'Generated chart preview',
    mediaType: 'image/svg+xml',
    data: base64Payload, // raw base64; an already-assembled data: URI also renders
  },
];

<ActivityImages images={images} />
```
