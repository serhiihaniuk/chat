Message is the leaf that renders one turn of conversation text. There is no Base UI part:
a turn is a semantic `<div data-from>` whose role drives layout and skin entirely through
the `data-[from=user]:` variant, so the **same** component renders both sides.

A `user` turn is a right-aligned bubble with one squared tail corner (`rounded-br-sm`),
capped at 82% of the column so long lines wrap before the edge. Its fill resolves through
`--message-user` to the shared active surface (`--row-bg-active, falling back to --accent`),
and its text resolves through `--message-user-foreground` to `--foreground`. An
`assistant` turn has no bubble: it is full-measure prose rendered through `MarkdownContent`
and capped at the reading measure (`--size-measure-message`). The 82% cap is an inline
style (a runtime value, not a class); leading is the registered `leading-message` token.

## Usage

```tsx
import { Message } from '@side-chat/side-chat-widget/ui/message';

<Message role="user" text="How do I memoize an expensive value?" />
<Message role="assistant" mode="streaming" text="Use the `useMemo` hook..." />
```

`role` selects the layout: `user` paints the bubble (`--message-user` /
`--message-user-foreground`, resolving to `--row-bg-active, falling back to --accent` /
`--foreground`),
`assistant` renders
Markdown. Pass `mode="streaming"` while tokens are still arriving so the Markdown wrapper
renders incrementally.

`MarkdownContent` is the single owner of the `.sc-markdown` hook. Streamdown owns the
generated elements under that hook; links, code, lists, headings, quotes, and tables consume
the message tokens below instead of raw spacing, color, or typography utilities. This keeps
legacy and workflow messages on the same theme and density contract.

Ordered lists use a wider gutter than bullets. The dedicated
`--message-ordered-list-indent` token keeps three-digit markers such as `120.` inside the
message measure, while `--message-list-indent` preserves the tighter bullet-list rhythm.

Headings inside chat answers are intentionally compact, not document-sized. `h1` resolves to
`--message-heading-1-font-size`, `h2` to `--message-heading-2-font-size`, and `h3` through `h6`
to `--message-heading-3-font-size`. Streamdown component overrides own those classes so its
default `text-3xl`/`text-2xl` utilities cannot escape the widget token contract. Their larger
before/after gaps preserve section rhythm without making the heading type itself oversized.

Fenced code is a Streamdown composite, so `.sc-markdown` owns its stable `data-streamdown`
hooks instead of depending on Streamdown's internal utility classes. The frame, header, action
overlay, and body use message tokens; the nested `pre` is reset so it cannot create a second
padded card. Streamdown's fixed 200px virtualization placeholder is replaced by
`--message-code-block-placeholder-height`, preserving off-screen rendering performance while
letting Compact, Cozy, and Roomy density scale the reserved space.

Code blocks expose Copy only. File download is intentionally disabled because the same content is
already available through Copy, while a second adjacent action crowded the compact message header.
The Copy glyph uses the shared compact `--size-icon-sm` tier; completed-message actions use the
same tier, so third-party defaults cannot render a 24px icon beside 14px chat text.
