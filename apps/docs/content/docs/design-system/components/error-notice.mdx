The error notice marks a turn that failed to complete. It sits on a `--muted` surface inside a `--border` outline - never a full-red panel. The only destructive color is the leading `TriangleAlert` glyph, tinted through the `sc-error-glyph` hook (`--destructive` mixed into `--muted-foreground`, since there is no destructive-foreground). Error, blocked, cancelled, and truncated notice glyphs all use the shared compact `--size-icon-sm` tier. Body copy stays tier-1 `--foreground`. When an `onRetry` handler is supplied, the shared secondary "Try again" button appears on `--secondary`, which aliases `--muted`; hover still reads `--accent`, and pressing it re-runs the same turn.

## Usage

```tsx
import { ErrorNotice } from '@side-chat/side-chat-widget/ui/error-notice';

<ErrorNotice
  message="The model failed to respond. Check your connection and try again."
  onRetry={() => regenerate()}
/>
```
