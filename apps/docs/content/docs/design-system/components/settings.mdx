Settings is a responsive `Tabs.Root` where one group selection drives **both** navigators and the
same `Tabs.Panel` set. Wide layouts show a left rail (`Tabs.List`); narrow layouts keep the tabs
mounted but swap the navigator for a top `Select`. The flip is decided by the `.sc-settings-frame`
container query, so the panel re-flows on its own width rather than the viewport's. Inside, the
Theme group lays out theme cards, accent swatches, and `Segmented` controls for corners, density,
text size, typeface, and elevation; the General group holds the model picker, instructions field,
and a send-on-Enter `Switch`. Appearance controls re-skin the live surface by mutating the shared
root tokens every component reads: `--radius`, `--space-unit`, `--text-*`, `--font-widget`, and
`--shadow-*`.

## Usage

```tsx
import { SettingsPanel } from '@side-chat/side-chat-widget/ui/settings';

// Uncontrolled: the panel owns group, theme, appearance, and model.
<SettingsPanel />

// Controlled appearance — the host passes persisted values and applyAppearance={false}
// when an outer widget root already applies the appearance token overrides.
<SettingsPanel
  theme={theme}
  onThemeChange={setTheme}
  accent={accent}
  onAccentChange={setAccent}
  corners={corners}
  onCornersChange={setCorners}
  density={density}
  onDensityChange={setDensity}
  textSize={textSize}
  onTextSizeChange={setTextSize}
  typeface={typeface}
  onTypefaceChange={setTypeface}
  elevation={elevation}
  onElevationChange={setElevation}
  applyAppearance={false}
/>
```

`SettingsSection` wraps `SettingsPanel` in the framed header shell used by the showcase and the
preview above.
