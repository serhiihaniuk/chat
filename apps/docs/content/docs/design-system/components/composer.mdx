The Composer is the message input shell. A Base UI `Field` textarea sits over a controls row that hosts the `ToolsMenu` (`+`), a context meter, a model selector slot, and the send button. The focus ring lives on the whole surface via `sc-composer` (`:focus-within`), never on the raw textarea, so tabbing into the field lights the entire shell. Send is a single button that swaps idle / armed / stop through the `data-armed` state on the `sc-send` hook class — it shows an `ArrowUp` while composing and a `Square` while a response is `submitted` or `streaming`.

The context meter is a `role="meter"` that reports real token usage: pass `contextUsedTokens` (the last completed turn's tokens) and `contextWindowTokens` (the active model's window) and it fills accordingly, naming "used / window tokens" on hover. Omit either and it renders nothing — it never shows a fabricated value. The field stays editable while a turn streams so the next message can be drafted; with `sendOnEnter={false}` a bare Enter inserts a newline and Ctrl/Cmd+Enter sends.

## Usage

```tsx
import { Composer } from '@side-chat/side-chat-widget/ui/composer';

<Composer
  contextUsedTokens={156_000}
  contextWindowTokens={200_000}
  placeholder="Message Side Chat..."
  status="idle"
  onSubmit={(text) => send(text)}
  onStop={() => abort()}
/>;
```

Leave `modelSelector` unset to use the built-in `ModelSelector`, or pass your own node to swap the slot. Control the value with `value` + `onValueChange`, or let it manage its own text via `defaultValue`.
